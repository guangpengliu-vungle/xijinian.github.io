发表于 2018-05-14 Disqus：

什么是面向切面编程

AOP 术语

  • 通知（Advice）
    • 前置通知
    • 后置通知
    • 返回通知
    • 异常通知
    • 环绕通知
  • 连接点（Join point）
  • 切点（Pointcut）
  • 切面（Aspect）
  • 引入（Introduction）—— 在无需修改现有类的情况下，让它们具有新的行为和状态
  • 织入（Wearing）—— 把切面应用到目标对象并创建新的代理对象的过程
    • 编译器
      切面在目标类编译时被织入，需要特殊的编译器，AspectJ 的织入编辑器就是以这种方式织入切面
    • 类加载期
      切面在目标类加载到 JVM 时织入，这种方式需要特殊的 ClassLoader，它可以在目标类被引入应用之前增强该目标类的字节码。AspectJ 5 LTW（load-time weaving）就支持以这种方式织入切面
    • 运行期
      一般情况下，AOP 容器会为目标对象动态创建一个代理对象，这就是 Spring AOP

综述：连接点是程序执行过程中能够应用通知的所有点。通知定义了切面的时间和行为，切点定义了切面的位置（在哪些连接点）。

Spring 对 AOP 的支持

  1. 基于代理的经典 Spring AOP
  2. 纯 POJO 切面
  3. @AspectJ 注解驱动的切面
  4. 注入式 AspectJ 切面（适用于 Spring 各3版本）

以上前三种都是 Spring AOP 实现的变体，构建在动态代理的基础上，只能局限于方法拦截。当应用需要被代理的 Bean 时，Spring 才创建代理对象。如果使用的是 ApplicationContext 的话，在 ApplicationContext 从 BeanFactory 中加载所有 Bean 的时候，Spring 才会创建被代理的对象。

通过切点选择连接点

execution 表达式

  • modifiers：方法的可见性，如 public，protected
  • ret-type：方法的返回值类型，如 int，void
  • declaring-type：方法所在类的全路径名，如 com.spring.Aspect
  • name：方法名，如 buisinessService()
  • param：方法的参数类型，如 java.lang.String
  • throws：方法抛出的异常类型，如 java.lang.Exception

通配符

  • * 通配符主要用于匹配单个单词，或者是以某个词为前缀或后缀的单词
  • .. 通配符表示0个或多个项。
    用于 declaring-type 中，表示匹配当前包及其子包；用于 param 中，则表示匹配 0 个或多个参数

指示器

指示器 描述
execution() 用于匹配是连接点（静态匹配）
args() 匹配指定参数类型、指定参数数量的方法级的连接点（动态匹配）
@args() 匹配参数所属类型上被指定注解所标注的方法参数（不是方法参数中有注解）
this() 匹配指定类型的代理对象
target() 匹配指定类型的目标对象（被代理的对象）
@target() 匹配被指定注解所标注的类（动态匹配）
within() 匹配指定类型下的所有连接点，只接受类型声明（允许使用通配符）（静态匹配）
@within() 匹配被指定注解所标注的类（静态匹配）
@annotation() 匹配被指定注解所标注的方法（注解作用在方法上面）
bean() 限制连接点匹配 AOP 代理的 Bean 引用为指定的 Bean

AOP 常用标签

AOP 配置元素 用途
<aop:advisor> 定义 AOP 通知器
<aop:after> 定义 AOP 后置通知（不管被通知的方法是否执行成功）
<aop:after-returning> 定义 AOP 返回通知
<aop:after-throwing> 定义 AOP 异常通知
<aop:around> 定义 AOP 环绕通知
<aop:aspect> 定义一个切面
<aop:aspectj-autoproxy> 启用 @AspectJ 注解驱动的切面
<aop:before> 定义一个 AOP 前置通知
<aop:config> 顶层的AOP配置元素。
大多数的 <aop:*> 元素必须包含在 <aop:config> 元素内
<aop:declare-parents> 以透明的方式为被通知的对象引入额外的接口
<aop:pointcut> 定义一个切点

注入 AspectJ 切面

Spring 不能像之前那样使用 <bean> 声明来创建一个 CriticAspect 实例 —— 它已经在运行时由 AspectJ 创建完成了，幸好所有的 AspectJ 切面都提供了一个静态的 aspectOf() 方法，该方法返回切面的一个单例。Spring 需要通过 aspectOf() 工厂方法获得切面的引用，然后像 <bean> 元素规定的那样在该对象上执行依赖注入。
