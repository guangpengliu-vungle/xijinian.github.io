REST 基础知识

REST 和 RPC 关系不大：RPC 是面向服务的，并关注行为和动作；REST 是面向资源的。

RE[$Representational^{表述性}$]S[$State^{状态}$]T[$Transfer^{转移}$] —— 就是将资源状态以最适合客户端或服务端的形式从服务器端转移到客户端（或者反过来）。

在 REST 中，资源通过 URL 进行识别和定位，关注的是资源，而不是行为。REST 中会有行为，但不体现在 URL 中，而是通过 HTTP 方法定义的，和 CRUD 对应：

  • $POST^{Create}$
  • $GET^{Read}$
  • $PUT/PATCH^{Update}$
  • $DELETE^{Delete}$

Spring AOP IN ACTION

发表于 2018-05-14 Disqus：

什么是面向切面编程

AOP 术语

  • 通知（Advice）
    • 前置通知
    • 后置通知
    • 返回通知
    • 异常通知
    • 环绕通知
  • 连接点（Join point）
  • 切点（Pointcut）
  • 切面（Aspect）
  • 引入（Introduction）—— 在无需修改现有类的情况下，让它们具有新的行为和状态
  • 织入（Wearing）—— 把切面应用到目标对象并创建新的代理对象的过程
    • 编译器
      切面在目标类编译时被织入，需要特殊的编译器，AspectJ 的织入编辑器就是以这种方式织入切面
    • 类加载期
      切面在目标类加载到 JVM 时织入，这种方式需要特殊的 ClassLoader，它可以在目标类被引入应用之前增强该目标类的字节码。AspectJ 5 LTW（load-time weaving）就支持以这种方式织入切面
    • 运行期
      一般情况下，AOP 容器会为目标对象动态创建一个代理对象，这就是 Spring AOP

综述：连接点是程序执行过程中能够应用通知的所有点。通知定义了切面的时间和行为，切点定义了切面的位置（在哪些连接点）。

Spring MVC 起步

请求流程

请求的第一站是 DispatcherServlet，它就 Spring MVC 的前端控制器，它作为一个单实例的 Servlet 将请求委托给合适的控制器来执行实际的处理，而要查找合适的控制器就要 DispatcherServlet 查询一个或多个 handler mapping 来确定请求的下一站在在哪。handler mapping 会根据请求所携带的 URL 信息来进行决策。 一旦选择了合适的控制器，DispatcherServlet 会将请求发送给选中的控制器。到了控制器，请求会卸下其负载（用户提交的信息）并耐心等待控制器处理这些信息。 控制器处理完后将产生的信息（model）打包发回给 DispatcherServlet，同时传递一个视图（view）名。DispatcherServlet 将会使用视图解析器（view resolver）来将逻辑视图名匹配为一个特定的视图实现，最终渲染输出。

发表于 2018-05-10 Disqus：
  • 同步和异步：同步和异步通常用来形容一次方法调用。同步方法一旦调用开始，就得等调用返回，才能继续后续的行为。异步方法更像是一个消息传递，调用后方法立即返回，整个过程不会阻碍后续的工作，如果异步调用需要返回结果，那么当这个异步调用真实完成时，会通知调用者。
  • 并发和并行：并发是快速切换，快到感觉不到。并行需要硬件的支持。
  • 临界区：形容一种公共资源，可以被多个线程使用，但是每一次只能有一个线程使用它。
