MyBatis 最简化配置

  1. 第一步当然是制作全局 XML 文件，名字没什么要求，比如可以叫做——mybatis-config.xml，放在 src 目录下面：

    xxxxxxxxxx
    <?xml version="1.0" encoding="UTF-8" ?>
    <!DOCTYPE configuration
      PUBLIC "-//mybatis.org//DTD Config 3.0//EN"
      "http://mybatis.org/dtd/mybatis-3-config.dtd">
    <configuration>
        <environments default="development">
            <environment id="development">
                <transactionManager type="JDBC"/>
                <dataSource type="POOLED">
                    <property name="driver" value="${driver}"/>
                    <property name="url" value="${url}"/>
                    <property name="username" value="${username}"/>
                    <property name="password" value="${password}"/>
                </dataSource>
            </environment>
        </environments>
        <mappers>
            <!--这个 XML Mapper 下面就说-->
            <mapper resource="xyz/xijinian/mapper/BlogMapper.xml"/>
        </mappers>
    </configuration>

  2. 第二步，新建 xyz/xijinian/mybatisdemo/mapper/BlogMapper.xml：

    xxxxxxxxxx
    <?xml version="1.0" encoding="UTF-8" ?>
    <!DOCTYPE mapper
      PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN"
      "http://mybatis.org/dtd/mybatis-3-mapper.dtd">
    <mapper namespace="xyz.xijinian.mapper.BlogMapper">
        <select id="selectBlog" resultType="Blog">
            select * from Blog where id = #{id}
        </select>
    </mapper>

  3. 用 XML 文件构建 SqlSessionFactory，不过首先要先构建 SqlSessionFactoryBuilder：

    xxxxxxxxxx
    String resource = "mybatis-config.xml";
    InputStream inputStream = Resources.getResourceAsStream(resource);
    SqlSessionFactory sqlSessionFactory = new SqlSessionFactoryBuilder().build(inputStream);

  4. 用 SqlSessionFactory 生产 SqlSession：

    • 老方式

      xxxxxxxxxx
      SqlSession session = sqlSessionFactory.openSession();
      try {
          Blog blog = (Blog)session.selectOne("xyz.xijinian.mapper.BlogMapper.selectBlog", 101);
      } finally {
          session.close();
      }

    • 新方式

      xxxxxxxxxx
      SqlSession session = sqlSessionFactory.openSession();
      try {
          BlogMapper mapper = session.getMapper(BlogMapper.class);
          Blog blog = mapper.selectBlog(101);
      } finally {
          session.close();
      }

  5. 最佳实践

    • SqlSessionFactoryBuilder 实例的最佳作用域是方法作用域（也就是局部方法变量），最好不要让其一直存在以保证所有的 XML 解析资源开放给更重要的事情。

    • SqlSessionFactory 一旦被创建就应该在应用的运行期间一直存在，没有任何理由对它进行清除或重建。有很多方法可以做到，最简单的就是使用单例模式或者静态单例模式。

    • 每个线程都应该有它自己的 SqlSession 实例。SqlSession 的实例不是线程安全的，因此是不能被共享的，所以它的最佳的作用域是请求或方法作用域。每次收到的 HTTP 请求，就打开一个 SqlSession；返回一个响应，就关闭它。这个关闭操作是很重要的，可以在 finally 中关闭。

    • 映射器接口的实例是从 SqlSession 中获得的。因此从技术层面讲，任何映射器实例的最大作用域是和请求它们的 SqlSession 相同的。但实际上，映射器实例的最佳作用域是方法作用域：

      xxxxxxxxxx
      SqlSession session = sqlSessionFactory.openSession();
      try {
          BlogMapper mapper = session.getMapper(BlogMapper.class);
          // do work
      } finally {
          session.close();
      }

三种查询方式

  1. selectList() 返回值为 List 的泛型类型的对象

    • 对应的 xml

      xxxxxxxxxx
      <select id="selAll" resultType="xyz.xijinian.pojo.Flower">
          select id,name name123,price,production from flower
      </select>

    • 调用方式

      xxxxxxxxxx
      List<Flower> list = session.selectList("xyz.xijinian.mapper.FlowerMapper.selAll");
      for (Flower flower : list) {
          System.out.println(flower.toString());
      }

  2. selectOne() 返回值 Object 类型的对象

    • 对应的 xml

      xxxxxxxxxx
      <select id="calCount" resultType="int">
          select count(*) from flower
      </select>

    • 调用方式

      xxxxxxxxxx
      int count = session.selectOne("xyz.xijinian.mapper.FlowerMapper.calCount");
      System.out.println(count);

  3. selectMap() 返回值 Map

    • 对应的 xml

      xxxxxxxxxx
      <select id="getMap" resultType="xyz.xijinian.pojo.Flower">
          select id,name name123,price,production from flower
      </select>

    • 调用方式

      xxxxxxxxxx
      Map<Object, Object> map = session.selectMap("xyz.xijinian.mapper.getMap", "name123");
      System.out.println(map);

形式

@XXXX(属性名=值)

值的分类（如果注解只需要给一个属性赋值，且这个属性是默认属性，那么可以省略属性名）： ① 基本类型 —— @XXXX(属性名=值) ② 数组类型 —— @XXXX(属性名={值,值}) ③ 引用类型 —— @XXXX(属性名=@值)

路径

相对路径

从当前资源出发找到其他资源

绝对路径

从根目录（服务器根目录或项目根目录）出发找到其他资源（只要以 “/” 开头的都是绝对路径）

注意：如果是请求转发，“/” 表示项目根目录，其他重定向 <img/> 、<script/>、<style/>、location.href 等 “/” 都表示服务器根目录（tomcat/webapps 文件夹）。如果客户端请求的控制器转发到 JSP 后，JSP 中如果使用相对路径，则需要按照控制器的路径去找其他资源。

parameterType

简介

SqlSession 的 selectList() 、selectOne() 的第二个参数和 selectMap() 的第三个参数都表示方法的参数，也就是 parameterType，在 XxxMapper.xml 中可以通过 #{} 来获取参数

xxxxxxxxxx
<select id="selById" resultType="xyz.xijinian.pojo.People" parameterType="int">
    select * from people where id=#{0}
</select>

#{} 获取参数的方式： ① 使用索引，从 0 开始，#{0} 表示第一个参数 ② 使用 #{param1} 获取第一个参数 ③ 如果只有一个参数（基本数据类型或 String），MyBatis 对 #{} 里面内容没有要求，只要写内容即可 ④ 如果参数是对象，就用 #{属性名} （前提：对应的属性必须有 get/set 方法） ⑤ 如果参数是 map，就用写成 #{key}

#{} 和 ${} 的区别

  • #{} 获取参数的内容支持索引获取，也支持 param1 这种获取指定位置参数的形式，并且 SQL 语句使用 “?” 占位符
  • ${} 字符串拼接不使用 “?”，默认找 ${内容} 内容的 get/set 方法，如果内容是数字，就直接带入一个字面量

XML 文件中的转义字符

操作 XML 文件时，如果允许用户输入 "< "、">"、"/"、"" 等内容，当渲染 XML 时这些内容就会破坏 XML 的结构，为了防止此事，我们可以使用 [CDATA[%s]]，这其中的字符串不会被当做标签解析。

"?" 的陷阱

SQL 不允许在 "?" 关键字前后进行数学运算，如需计算可在 Java 代码中计算出结果后再传递到 XxxMapper.xml 中。

MyBatis 底层是对 JDBC 的封装，其默认是关闭了 JDBC 的自动提交功能

JDBC 中 executeUpdate() 执行新增、删除、修改的 SQL 返回值都是 int，表示受影响的行数 MyBatis 中 <insert>、<delete>、<update> 标签没有 resultType 属性，因为其认为返回值都是 int

在 openSession() 时 Mybatis 会在创建 SqlSession 同时创建一个 Transaction（事务对象），同时 autoCommit 都为 false，session.commit() 提交事务，openSession(true) 自动提交，即 .setAutoCommit(true)。如果出现异常，应该 session.rollback() 回滚事务

ThreadLocal

ThreadLocal 是线程容器，给线程绑定一个 Object 后，只要线程不变，可以随时取出它，但是线程变了，就无法获取：

xxxxxxxxxx
final ThreadLocal<String> = new ThreadLocal<>();
threadLocal.set("测试");
new Thread(){
    public void run() {
        String result = threadLocal.get();
        System.out.println("结果:" + result);
    }
}.start();

缓存

SqlSession 缓存

SqlSession 缓存又称为一级缓存，它是 MyBatis 默认开启的，当一个 SqlSession 第一次通过 SQL 和参数获取对象后，它就会将其缓存起来，如果下次的 SQL 和参数都没有发生变化，并且缓存没有超时或者声明需要刷新时，那么它就会从缓存中获取数据，而不是通过 SQL 获取了。

事实上，MyBatis 缓存的是 statement 对象（简单记忆必须是用一个 <select>），在 MyBatis 中一个 <select> 对应一个 statement 对象，以及缓存的有效范围必须是同一个 SqlSession 对象。

SqlSessionFactory 缓存

SqlSessionFactory 缓存又称为二级缓存，如果两次查询基于同一个 SqlSessionFactory，那么就从二级缓存中取数据，而不用到数据库里去取了。注意：只有 session.commit() 或者 session.close() 执行后，一级缓存才会刷到二级缓存。

配置步骤：

① 在 XXXMapper.xml 中添加 <cache /> 标签，同时 POJO 实现 Serializable 接口，并且要执行 session.commit() 或者 session.close()。

② 如果不想这么麻烦，可以直接在 XXXMapper.xml 中添加 <cache readOnly="true"></cache> 标签。

多表查询

使用 <resultMap> 标签时，<select> 标签不写 resultType 属性，而是使用 resultMap 属性引用 <resultMap> 标签。

单表映射

// POJO
public class Teacher {
    private int id1;
    private String name1;

}
xxxxxxxxxx
<!-- property 是映射到列结果的字段或属性 -->
<!-- column 对用的是 SQL 的列 -->
<resultMap type="teacher" id="mymap">

关联单个对象（N+1）

N+1 效率低，但是有时候你的业务不确定，这样分开写也不错

public class Student {
    private int id;
    private String name;
    private int age;
    private int tid;
    private Teacher teacher;
    // getter、setter...
}
xxxxxxxxxx
<!-- TeacherMapper.xml -->
<select id="selById" resultType="teacher" parameterType="int">
    select * from teacher where id=#{0}
</select>
xxxxxxxxxx
<!-- StudentMapper.xml -->
<resultMap type="student" id="stuMap">
    <id property="id" column="id"/>
    <result property="name" column="name"/>
    <result property="age" column="age"/>
    <result column="tid" property="tid"/>
    <!-- 如果关联一个对象 -->
    <association property="teacher" 
                 select="xijinian.xyz.mapper.TeacherMapper.selById" 
                 column="tid">
    </association>
</resultMap>
<select id="selAll" resultMap="stuMap">
    select * from student
</select>
​
<!-- 使用 N+1 方式时，利用 Auto mapping 特性，将上述代码简化为↑↓ -->
​
<resultMap type="student" id="stuMap">
    <result column="tid" property="tid"/>
    <association property="teacher" 
                 select="xijinian.xyz.mapper.TeacherMapper.selById" 
                 column="tid">
    </association>
</resultMap>
<select id="selAll" resultMap="stuMap">
    select * from student
</select>

Auto Mapping 多表查询

“.”在 SQL 中是关键字符，两侧需添加反单引号。要求：只能使用多表联合查询方式，查询出的列名和要和属性名相同，且只能关联单个对象，而不能关联集合。

xxxxxxxxxx
<!-- StudentMapper.xml -->
<mapper namespace="com.bjsxt.mapper.StudentMapper">
    <select id="selAll" resultType="student">
        select 
            t.id `teacher.id`,
            t.name `teacher.name`,
            s.id id,
            s.name name,
            age,
            tid 
        from student s LEFT JOIN teacher t on t.id=s.tid
    </select>
</mapper>
xxxxxxxxxx

MyBatis 运行原理

在 MyBatis 运行开始时需要先通过 Resources 加载全局配置文件，接着需要实例化 SqlSessionFactoryBuilder 构建器，帮助 SqlSessionFactory 接口实现类 DefaultSqlSessionFactory，在实例化 DefaultSqlSessionFactory 之前需要先创建 XmlConfigBuilder 解析全局配置文件流，并把解析结果存放在 Configuration 中。之后把 Configuration 传递给 DefaultSqlSessionFactory，到此 SqlSessionFactory 工厂创建成功。 由 SqlSessionFactory 工厂创建 SqlSession，每次创建 SqlSession 时，都需要由 TransactionFactory 创建 Transaction 对象，同时还需要创建 SqlSession 的执行器 Excutor，最后实例化 DefaultSqlSession，传递给 SqlSession 接口。根据项目需求使用 SqlSession 接口中的 API 完成具体的事务操作。
