Ioc

作用

Spring 框架环境搭建

  1. 导入四个核心 JAR 包

  2. 在 src 下面新建 applicationContext.xml

    • 文件名和路径其实可以自定义

    • applicationContext.xml 文件中配置的信息最终存储到了 Spring 容器 ApplicationContext 中

    • 配置文件需要引入 schema

      • schema 文件的拓展名是 .xsd
      • 比 DTD 具有更好的拓展性
      • 每次引入一个 .xsd 文件就是引入一个 namespace

  3. 通过 applicationContex.xml 文件创建对象：

    • 通过 <bean .../> 指定对象

      ​x
      <?xml version="1.0" encoding="UTF-8"?>
      <beans xmlns="http://www.springframework.org/schema/beans" 
          xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
          xsi:schemaLocation="http://www.springframework.org/schema/beans
              http://www.springframework.org/schema/beans/spring-beans.xsd">
      ​
          <bean id="peo" class="xyz.xijinian.pojo.People"/>
      </beans>

    • 通过 Spring 实例化对象：

      xxxxxxxxxx
      public static void main(String[] args) {
          ApplicationContext ac = new 
              ClassPathXmlApplicationContext("applicationContext.xml");
          // 如果在 getBean() 方法中没有第二个参数，就会返回 Object 类型
          People people = ac.getBean("peo", People.class);
          System.out.println(people);
      }

Spring 创建对象的三种方式

  1. 通过构造方法创建对象

    • 通过无参构造方法创建（即上文中的那种）

    • 通过有参构造方法创建

      • 需要在 POJO 中提供类的有参构造方法

      • 在 applicationContext.xml 文件中配置

        xxxxxxxxxx
        <?xml version="1.0" encoding="UTF-8"?>
        <beans xmlns="http://www.springframework.org/schema/beans" 
            xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" 
            xsi:schemaLocation="http://www.springframework.org/schema/beans
                http://www.springframework.org/schema/beans/spring-beans.xsd">
            
            <bean id="peo" class="xyz.xijinian.pojo.People">
                <!-- 默认匹配最后一个符合条件的构造方法，可通过 name、type 精确指定 -->
                <constructor-arg index="0" value="123"></constructor-arg>
                <constructor-arg index="1" value="张三"></constructor-arg>
            </bean>
        </beans>

  2. 通过实例工厂创建对象

    • 首先需要有一个实例工厂

      xxxxxxxxxx

              ClassPathXmlApplicationContext("applicationContext.xml");
      People people = ac.getBean("peo1", People.class);

AOP

作用

在程序原有的纵向执行流程中，针对某一个或某一些方法添加通知，形成切面过程就叫做面向切面编程。它可以无侵入拓展原来代码的功能，非常灵活。

常用概念

  • 切点（Pointcut）：原有功能
  • 前置通知（before advice）：在切点之前执行的功能
  • 后置通知（after advice）：在切点之后执行的功能
  • 异常通知（throws advice）：切点执行过程中出现异常
  • 切面：所有功能的总称
  • 织入：把切面嵌入到原有功能的过程就叫做织入

实现 AOP 的两种方式

  1. Schema-based

    • 每个通知都需要实现接口或类

    • 配置 applicationContext.xml 文件时在 <aop:config> 这个标签里面配置

    • 实现步骤

      1. 导入相关 JAR 包

      2. 写要被织入通知的类

        xxxxxxxxxx
        public class Demo {
            public void demo1 {
                System.out.println("demo1");
            }
            public void demo2 {
                System.out.println("demo2");
            }
            public void demo3 {
                System.out.println("demo3");
            }
        }

      3. 建立前置通知以及后置通知类

        xxxxxxxxxx
        // 建立前置通知类
        public class MyBeforeAdvice implements MethodBeforeAdvice {
            @Override
            // arg0：切点方法对象
            // arg1：切点方法参数
            // arg2：切点在哪个对象中
            public void before(Method arg0, Object[] arg1, Object arg2) 
                throws Throwable {
                System.out.println("执行前置通知");
            }
        }
        ​
        // 建立后置通知类
        public class MyAfterAdvice implements AfterReturningAdvice {
            @Override
            // arg0：切点方法返回值
            // arg1：切点方法对象
            // arg2：切点方法参数
            // arg3：切点方法所在类的对象
            public void afterReturning(Object arg0, Method arg1, Object[] arg2, 
                                       Object arg3) throws Throwable {
                System.out.println("执行后置通知");
            }
        }

      4. 配置 applicationContext.xml

        xxxxxxxxxx
        <?xml version="1.0" encoding="UTF-8"?>
        <beans xmlns="http://www.springframework.org/schema/beans" 
            xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" 
            xmlns:aop="http://www.springframework.org/schema/aop" 
            xsi:schemaLocation="http://www.springframework.org/schema/beans
            http://www.springframework.org/schema/beans/spring-beans.xsd
            http://www.springframework.org/schema/aop
            http://www.springframework.org/schema/aop/spring-aop.xsd">
            
            <!-- 配置通知类对象，在切面中引入 -->
            <bean id="mybefore" class="xyz.xijinian.advice.MyBeforeAdvice"></bean>
            <bean id="myafter" class="xyz.xijinian.advice.MyAfterAdvice"></bean>
            
            <!-- 配置切面 -->
            <aop:config>
                <!-- 配置切点 -->
                <aop:pointcut expression="execution(* xyz.xijinian.test.Demo.demo2())" id="mypoint"/>
                <!-- 通知 -->
                <aop:advisor advice-ref="mybefore" pointcut-ref="mypoint"/>
                <aop:advisor advice-ref="myafter" pointcut-ref="mypoint"/>
            </aop:config>
            <!-- 配置 Demo 类，测试使用 -->
            <bean id="demo" class="xyz.xijinian.test.Demo"></bean>
        </beans>
        ​
        <!-- 在配置切点那里，第一个"*"声明通配符，如果方法中有参数，可以这样写： -->
        <!-- expression="execution(* xyz.xijinian.test.Demo.*(..))" -->

      5. 测试代码

        xxxxxxxxxx
        public class Test {
            public static void main(String[] args) {
                ApplicationContext ac = 
                    new ClassPathXmlApplicationContext("applicationContext.xml");
                Demo demo = ac.getBean("demo", Demo.class);
                demo.demo1();
                demo.demo2();
                demo.demo3();
            }
        }

  2. AspectJ

    • 每个通知不需要实现接口或类
    • 配置 applicationContext.xml 文件时在 <aop:config> 这个标签的子标签 <aop:aspect> 里面配置

配置异常通知（AspectJ）

  1. 写一个正常的类，将其中一个方法作为切点

    xxxxxxxxxx
    // 不抛出异常也可以，就是异常通知类会获取不到异常信息
    public class Demo throws Exception{
        public void demo1() {
            int i = 5 / 0;
            System.out.println("demo1");
        }
    }

  2. 定义一个异常通知类，无需实现接口，方法的名称也随意

    xxxxxxxxxx
    public class MyThrowAdvice {
        public void myexception(Exception e) {
            System.out.println("执行异常通知" + e.getMessage());
        }
    }

  3. 配置 applicationContext.xml

    xxxxxxxxxx
    <!-- <aop:aspect> 的 ref 属性表示异常方法在哪个类中
    <!-- <aop:xxxx 表示表示什么通知
    <!-- method 表示当触发这个通知时，调用哪个方法
    <!-- throwing 表示对象名，必须和通知中方法参数名相同（可以不在通知中声明异常对象） -->
    ​
    <bean id="demo" class="xyz.xijinian.test.Demo"></bean>
    <bean id="mythrow" class="xyz.xijinian.advice.MyThrowAdvice"></bean>
    <aop:config>
        <aop:aspect ref="mythrow">
            <aop:pointcut expression="execution(* xyz.xijinian.test.Demo.demo1())" id="mypoint"/>
            <aop:after-throwing method="myexception" pointcut-ref="mypoint" throwing="e1" throwing="e"/>
        </aop:aspect>
    </aop:config>

  4. 写测试类

    xxxxxxxxxx
    public class Test {
        public static void main(String[] args) {
            Application application = 
                new ClassPathXmlApplicationContext("applicationContext.xml");
            Demo demo = application.getBean("demo", Demo.class);
            try {
                demo.demo1();
            } catch (Exception e) {}
        }
    }

环绕通知（Schema-based）

定义

就是把前置通知和后置通知都写到一个通知中

实现步骤

  1. 大部分与上诉内容差不多

  2. 定义一个环绕通知类，要继承 MethodInterceptor，并且要实现 invoke 方法

    xxxxxxxxxx
    public class MyArround implements MethodInterceptor {
        @Override
        public Object invoke(MethodInvocation arg0) throws Throwable {
            System.out.println("环绕-前置");
            // 放行，调用切点方式
            Object result = arg0.proceed();
            System.out.println("环绕-后置");
            return result;
        }
    }

  3. 配置 applicationContext.xml

    xxxxxxxxxx
    <bean id="demo" class="xyz.xijinian.test.Demo"></bean>
    <bean id="myarround" class="xyz.xijinian.advice.MyArround"></bean>
    <aop:config>
        <aop:pointcut expression="execution(* xyz.xijinian.test.Demo.demo1())" id="mypoint"/>
        <aop:advisor advice-ref="myarround" pointcut-ref="mypoint" />
    </aop:config>

    public class SimpleJDKDynamicProxyDemo {
    ​
        static interface IService {
            public void sayHello();
        }
    ​
        static class RealService implements IService {
            @Override
            public void sayHello() {

        static class SimpleInvocationHandler implements InvocationHandler {
            private Object realObj;
    ​
            public SimpleInvocationHandler(Object realObj) {
                this.realObj = realObj;
            }
    ​
            @Override
            // proxy 是代理，method 是真实对象的方法，args 是真实对象的方法参数
            public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
                System.out.println("entering " + method.getName());
                // 真实对象的方法.invoke(真实的对象, 真实对象的方法参数)
                Object result = method.invoke(realObj, args);
                System.out.println("leaving " + method.getName());
                return result;
            }
        }
    ​
        public static void main(String[] args) {
            IService realService = new RealService();
            // java.lang.reflect 包中的静态类 Proxy 的 newProxyInstance 方法有三个参数
            // ① 类加载器；②真实对象的所有的接口；③一个 InvocationHandler 接口类型（与 Proxy 同包）
            // InvocationHandler 接口只定义了一个方法 invoke，对代理接口所有方法的调用都会转给该方法
            IService proxyService = (IService)Proxy.newProxyInstance(
                IService.class.getClassLoader(),
                new Class<?>[] { IService.class }, 
                new SimpleInvocationHandler(realService)
            );
            proxyService.sayHello();
        }
    }

    public class SimpleCGLibDemo {
        static class RealService {
            public void sayHello() {
                System.out.println("hello");
            }
        }
    ​
        static class SimpleInterceptor implements MethodInterceptor {
            @Override
            // object —— 代理对象
            // method —— 真实对象的方法
            // args —— 真实对象方法的参数
            // proxy —— 代理对象重写父类的同名方法
            public Object intercept(Object object, Method method,
                    Object[] args, MethodProxy proxy) throws Throwable {
                System.out.println("entering " + method.getName());
                // 代理对象的方法.invokeSuper(代理对象, 真实对象方法的参数)
                Object result = proxy.invokeSuper(object, args);

            }

            enhancer.setCallback(new SimpleInterceptor());
            return (T) enhancer.create();
        }

声明式事务属性解释

xxxxxxxxxx
<tx:advice id="txAdvice" transaction-manager="txManager">
    <tx:attributes>
        <tx:method name="ins*" />
        <tx:method name="del*" />
        <tx:method name="upd*" />
        <tx:method name="*" readonly="true"/>
    </tx:attributes>
</tx:advice>

  1. name：表示需要有事务控制的方法，其中的字符串支持“*”通配符

  2. readonly：是否为只读事务（区别提交事务），提交事务在数据库的内部可能需要有一些事务锁之类的机制，速度比较慢。如果将事务设置为只读事务，数据库内部可能会自动优化，如果是查询的功能，建议使用只读事务。如果不写，readonly="false" 是默认值，比较适合增删改的操作。

  3. propagation：控制事务传播行为——当一个具有事务控制的方法被另一个有事务控制的方法调用后，需要如何管理事务（新建事务？在事务中执行？事务挂起？报异 常？）

    • REQUIRED（默认值），在当前的事务中执行，如果当前没有事务，就新建一个事务。
    • SUPPORTS，在当前的事务中执行，如果当前没有事务，就在非事务状态下执行。
    • MANDATORY，必须在事务内部执行，如果当前有事务，就在事务中执行，如果没有事务，报错。
    • REQUIRES_NEW，必须在事务中执行，如果当前没有事务，新建事务，如果当前有事务，把当前事务挂起。
    • NOT_SUPPORTED，必须在非事务下执行，如果当前没有事务，正常执行，如果当前有事务，把当前事务挂起。
    • NEVER，必须在非事务状态下执行，如果当前没有事务，正常执行，如果当前有事务，报错。
    • NESTED：必须在事务状态下执行，如果没有事务，新建事务，如果当前有事务，创建一个嵌套事务（虽然一个事务包含了另一个事务，但是如果执行完出错也会直接回滚到事务最初点）。

  4. isolation：事务隔离级别——在多线程或并发访问下如何保证访问到的数据具有完整性。

    • 下面强调几个概念

      • 脏读：事务 B 修改数据库→事务 A 读取事务 B 修改过的值→事务 B 提交失败回滚→事务 A 读取到了脏数据。

        解决办法：一个事务修改数据的时候另一个事务不可以进行读取。

      • 不可重复读：事务 A 读取数据→事务 B 修改数据并提交→事务 A 再读数据，发现两次读取的数据不一样。

        解决办法：一个事务读取数据的时候另一个事务不可以进行修改。

      • 幻读：事务 A 读取数据→事务 B 插入（或删除）数据并提交→事务 A 再读数据，发现之前读少（多）了一行。

        解决办法：一个事务读取和修改数据的时候另一个事务不可以插入和删除数据。

    • isolation 的属性

      • DEFAULT（默认）：由底层数据库自动判断应该使用什么隔离级别。
      • READ_UNCOMMITTED：可以读取未提交数据，可能出现脏读、不可重复读、幻读，当然这效率最高。
      • READ_COMMITTED：只能读取其他事务已提交数据，可以防止脏读，可能出现不可重复读和幻读。
      • REPEATABLE_READ：读取的数据被添加锁，防止其他事务修改此数据，可以防止脏读和不可重复读，可能出现幻读。
      • SERIALIZABLE：排队操作，对整个表添加锁。一个事务在操作数据时，另一个事务等待事务操作完成后才能操作这个表。最安全的，效率也是最低。

  5. rollback-for：异常类型全限定路径——当出现什么异常时需要进行回滚？

    • 建议给定该属性值
    • 手动抛异常一定要给该属性值.

  6. no-rollback-for：当出现什么异常时不滚回事务

常用注解

  • @Component 创建类对象，相当于配置<bean>

  • @Service 与 @Component 功能相同，写在 ServiceImpl 类上

  • @Repository 与 @Component 功能相同，写在数据访问层类上

  • @Controller 与 @Component 功能相同，写在控制器类上

  • @Resource（不需要写对象的 get/set）

    • JDK 中的注解
    • 默认按照 byName 注入，如果没有名称对象，按照 byType 注入，建议把对象名称和 spring 容器中对象名相同

  • @Autowired（不需要写对象的 get/set）

    • Spring 中的注解
    • 默认按照 byType 注入

  • @Value() 获取 properties 文件中内容

  • @Pointcut() 定义切点

  • @Aspect() 定义切面类

  • @Before() 前置通知

  • @After 后置通知

  • @AfterReturning 后置通知（必须切点正确执行）

  • @AfterThrowing 异常通知

  • @Arround 环绕通知
