限定通配符的局限

  • 需要写元素的时候，一般不用通配符

    ? 就是表示类型安全无知，? extends Number 表示是 Number 的某个子类型，但不知道具体子类型，如果允许写入，Java 就无法确保类型安全性，所以干脆禁止。

  • 参数间存在依赖关系时，一般也不用通配符

  • 返回值依赖于类型参数，一般也不用通配符

基本类型不能用于实例化类型参数

泛型类型参数不能用于静态变量和方法

静态变量和方法都是类型的属性，且与类型参数无关，所以不能使用泛型类类型参数（实参）。

不过，对于静态方法，它可以是泛型方法，可以声明自己的类型参数，这个参数与泛型类的类型参数是没有关系的。
